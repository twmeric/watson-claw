## 1. 產品概述

### 1.1 願景陳述

> Watson Claw 是專為香港大學生設計的 WhatsApp AI 助手，讓學生通過自然對話管理學業生活，無需下載 App，無需複雜設置，說話即可安排一切。

### 1.2 問題陳述

**大學生面臨的痛點：**

| 痛點 | 現有解決方案 | 問題 |
|-----|------------|------|
| 課業繁多難以追蹤 | 手機備忘錄 | 容易遺忘，沒有提醒 |
| 社交活動協調困難 | WhatsApp 群組 | 時間難以統一，經常撞期 |
| 作業截止日期混亂 | Google Calendar | 需要手動輸入，門檻高 |
| 生活瑣事繁雜 | 多款 App | 分散在不同地方，難以管理 |

### 1.3 解決方案

**Watson Claw = WhatsApp + AI + 自動化管理**

```
用戶說："明天下午3點有組會"
    ↓
Watson 理解意圖 → 解析時間 → 設置提醒 → 同步日曆
    ↓
用戶收到確認："✅ 已設置明天 15:00 的組會提醒"
```

### 1.4 核心價值主張

- **零下載**: 無需安裝 App，直接在 WhatsApp 使用
- **零學習成本**: 自然語言對話，說話就像跟朋友聊天
- **零配置**: 自動創建賬戶，漸進式收集信息
- **全自動**: AI 理解意圖，自動執行操作

## 4. 用戶故事

### 4.1 Epic 1: 創建提醒

**US-001: 創建簡單提醒**
```
作為: 大學生
我想: 通過 WhatsApp 創建提醒
使得: 我不會忘記重要事情

驗收標準:
- 可以說"明天下午3點有組會"
- Watson 理解時間和任務
- 收到確認回覆
- 到時間收到 WhatsApp 提醒
```

**US-002: 創建帶提前提醒的事件**
```
作為: 大學生
我想: 設置提前 15 分鐘提醒
使得: 我有時間準備

驗收標準:
- 默認提前 15 分鐘提醒
- 可以說"提前30分鐘提醒我"
- 收到包含提醒時間的確認
```

### 4.2 Epic 2: 管理日程

**US-003: 查詢今日日程**
```
作為: 大學生
我想: 問 Watson "今天有什麼安排"
使得: 我知道今天的計劃

驗收標準:
- 返回今天的所有提醒
- 按時間排序
- 顯示已完成和待完成
```

**US-004: 取消提醒**
```
作為: 大學生
我想: 取消已設置的提醒
使得: 活動取消時不會收到錯誤提醒

驗收標準:
- 說"取消組會提醒"
- Watson 找到並取消
- 收到確認回覆
```

### 4.3 Epic 3: 智能體驗

**US-005: 理解模糊時間**
```
作為: 大學生
我想: 說"後天中午"而不是具體時間
使得: 對話更自然

驗收標準:
- 理解"明天"、"後天"、"下週三"
- 理解"早上"、"中午"、"晚上"
- 正確解析為具體時間
```

**US-006: 記憶我的習慣**
```
作為: 大學生
我想: Watson 記得我每週三有組會
使得: 下次只需說"組會"

驗收標準:
- 檢測到重複模式
- 下次提到時主動確認
- 減少用戶輸入
```

## 5. 產品需求

### 5.1 功能需求 (FR)

| ID | 需求 | 優先級 | 驗收標準 |
|----|------|--------|---------|
| FR-001 | 接收 WhatsApp 消息 | P0 | 通過 SaleSmartly Webhook 接收 |
| FR-002 | 解析自然語言時間 | P0 | 支持 10+ 種時間表達 |
| FR-003 | 創建提醒記錄 | P0 | 存入 D1 數據庫 |
| FR-004 | 發送 WhatsApp 消息 | P0 | 通過 CloudWapi 發送 |
| FR-005 | 定時檢查提醒 | P0 | Cron 每小時執行 |
| FR-006 | 查詢用戶日程 | P1 | 支持今日/明日/全部查詢 |
| FR-007 | 取消提醒 | P1 | 按關鍵詞或 ID 取消 |
| FR-008 | Google Calendar 同步 | P2 | OAuth 授權，雙向同步 |
| FR-009 | 重複事件 | P2 | 支持 daily/weekly/monthly |
| FR-010 | 用戶偏好學習 | P3 | 記錄並應用常用設置 |

### 5.2 技術約束

```
必須使用:
- Cloudflare Workers (Serverless)
- D1 SQLite 數據庫
- SaleSmartly (WhatsApp 接收)
- CloudWapi (WhatsApp 發送)
- Deepseek API (MVP 階段 LLM)

限制:
- Worker 執行時間 < 50ms (cold start)
- D1 查詢時間 < 100ms
- 總響應時間 < 3 秒
```

### 5.3 用戶界面需求

**對話界面 (WhatsApp):**

```
【創建提醒】
用戶: 明天下午3點有組會
Watson: 
✅ 已為你設置提醒！

📋 組會
⏰ 明天 15:00
🔔 提醒：14:45

我會準時通知你 👍

【查詢日程】
用戶: 我今天有什麼安排？
Watson:
📅 今天的安排：

1. 15:00 - 組會
2. 18:00 - 健身房
3. 20:00 - 和朋友吃飯

加油！💪

【提醒通知】
Watson:
⏰ 提醒

📋 組會
⏰ 15:00 (還有15分鐘)

準備好出發了嗎？
```

### 5.4 數據需求

**用戶數據:**
- phone (主鍵)
- name, university, major
- preferences (JSON)
- created_at, last_active_at

**提醒數據:**
- id, user_phone
- task_content, reminder_time
- status (pending/sent/cancelled)
- created_at

**對話歷史:**
- id, user_phone
- message, direction
- intent, created_at

## 6. 非功能需求

### 6.1 性能需求

| 指標 | 目標 | 測量方法 |
|-----|------|---------|
| 消息響應時間 | < 3 秒 | 從接收到回覆 |
| Webhook 處理時間 | < 2 秒 | 不含發送 |
| 提醒準時性 | ± 1 分鐘 | Cron 執行頻率 |
| 系統可用性 | 99.9% | 每月統計 |

### 6.2 安全需求

| 需求 | 描述 |
|-----|------|
| SEC-001 | Webhook 必須驗證 Security Code |
| SEC-002 | API Keys 必須存儲在環境變量 |
| SEC-003 | 用戶數據隔離，不能互相訪問 |
| SEC-004 | 手機號格式化存儲，去除敏感信息 |
| SEC-005 | 對話歷史保留 90 天 |

### 6.3 可擴展性

```
目標規模:
- 1,000 用戶 (MVP)
- 10,000 用戶 (6個月)
- 100,000 用戶 (12個月)

擴展策略:
- Cloudflare Workers 自動擴展
- D1 支持最大 500GB
- 需要時遷移到 Postgres
```

### 6.4 監控需求

```
必須監控:
- Webhook 接收成功率
- LLM API 響應時間
- 提醒發送成功率
- 活躍用戶數 (DAU/MAU)

告警條件:
- 錯誤率 > 5%
- 響應時間 > 5 秒
- 服務不可用
```
